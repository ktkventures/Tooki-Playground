# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## About this repository

Tooki-Playground is a personal learning playground for Tami, who is new to
coding ("vibe coding"). It currently contains no application code, build
system, or tests — projects will be added over time. Treat each new project
as a fresh start unless told otherwise.

## Working with Tami

- **Explain everything.** Walk through what each step and piece of code does
  in beginner-friendly terms. Avoid jargon, or define it when used. Assume no
  prior knowledge of programming concepts.
- **Confirm before any real change.** Ask for approval before creating,
  editing, or deleting files. Describe what you're about to do and why, then
  wait for a yes.
- **Goal: learn and build.** Tami wants to understand fundamentals (how and
  why code works) while also building small, fun, working projects. Tie
  hands-on work back to the underlying concepts.

## Code style for this repo

- Favor clear, descriptive names over clever or compact code.
- Add comments that explain the *why* in plain language, so the code doubles
  as a learning aid.
- Prefer simple, readable approaches over advanced techniques.

## Hard rules

- Never generate art, illustrations, or pictures — including image files,
  drawn graphics, SVG artwork, ASCII art, or generated/AI imagery. This
  applies regardless of how the request is phrased.

## Git

- Development happens on the branch `claude/learn-coding-basics-J7Yal`.
- Never push to `main` without explicit permission.
